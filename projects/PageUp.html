Project PageUp

Exploring swipe-based interaction for long-form mobile reading

Problem

Reading long-form text on mobile devices often feels cognitively heavy, especially for short, frequent reading sessions. Existing e-book readers are optimized for page-based or scroll-based interactions, which don’t align well with mobile-native gesture habits shaped by short-form media.

The core question was not about content quality or attention span, but about interaction design.

Hypothesis

Primary hypothesis:
Long-form text may feel less cognitively heavy on mobile if content is broken into discrete chunks and navigated via familiar vertical swipe gestures.

Counter-hypothesis:
Chunking could disrupt narrative flow or feel artificial for deep readers.

MVP scope

  • Vertical swipe-based navigation
  • Mobile Safari compatibility
  • Preloaded clean Chapter 1 of The Great Gatsby
  • Optional EPUB upload
  • Variable text density
  • Reading progress indicators
  • Basic resume state
  • Session-level instrumentation

Key product decisions

  • Swipe-based navigation instead of scroll or pagination
  • Text density treated as a product variable
  • Sensible defaults before configuration
  • Controlled content input to preserve signal
  • Explicit session boundaries for analytics

Instrumentation

  • Session start and end time
  • Device type
  • Pages read and last page index
  • Progress percentage
  • Total and average reading time
  • Text density usage

Learnings (pre-launch)

  • Separating the experiment from the solution prevents scope drift
  • Defaults materially shape behavior
  • Early metric definition clarifies learning boundaries

Next steps

  • Release to users and analyze reading behavior
  • Refine default density based on usage
  • Evolve resume state only if repeat usage emerges
  • Improve EPUB normalization only if needed
